## Build Notes:
The vnap:latest Docker image was built in an Ubuntu 24.04 VM running in VMware on a MacBook Pro.
By default the MacOS filesystem is not case-sensitive. There are filenames in the vanetza-nap repo
that only differ by case. Therefore development directly on MacOs was not possible.
Once the docker image is built, it should be able to run in any Docker environment.

The scripts that build and run vanetza-nap simulations can be hosted in the MacOS filesystem and mounted
in the VM - which is useful for automatically syncing changes to the host file system.
Otherwise it's probably easier to just clone this repo directly into the VM.

## Build Instructions:
#### 1: MacOS setup w/ VMware Ubuntu 24.04 VM:

 - Config option 1: Clone this repo in the Linux VM
   - Run: git clone

 - Config option 2: Clone this repo to the host fs and mount its directory in VM
   - Share the cloned repo directory in the VM settings
   - scp 0-vmware-mount-dir.sh into the Linux VM
   - Run the script to mount it at the /mnt/hgfs mount point.
   - You can create soft links to the mounted directory for convenience.

#### 2. Clone vanetza-nap repo in VM
  - Clone the repo in the Linux VM in the user's home directory.
  - The vanetza-nap repo must be cloned into a filesystem that is case-senstive.<br>

#### 3. Build vanetza-nap Docker image (vnap:latest)
  - Run ./docker-build.sh script. Initial build takes ~20 minutes.<br>
  - The build script copies modified files into their appropriate locations in the venetza-nap directory structure.
  - If you provide any argument to the build script, it will copy original vanetza-nap source files instead.

#### 4. Run the simulation
  - Edit docker-compose.yaml to change certificates/keys used by RSU/OBU containers.
    - vnap-certs/certify contains root/aa/at certs & keys generated by the vanetza-nap 'certify' tool.
    - vnap-certs/c-its-pki contains root/aa/at certs & keys generated with the C-ITS-PKI utility.
  - Run ./start-vnap.sh to bring up the RSU and OBU containers.
  - It will bring up the log for the RSU container to monitor execution.
